## 2025-05-01 - Type Confusion XSS Bypass via Missing Recursive Sanitization
**Vulnerability:** The API endpoint `functions/api/quote.ts` used a top-level iteration approach to sanitize JSON requests. Its `sanitizeObject` wrapper only sanitized direct object properties and failed to traverse nested arrays or objects. By passing an array of strings like `["<script>alert(1)</script>"]` as a property, the inner string values completely bypassed HTML entity encoding. When these unsanitized payloads were interpolated into HTML template literals later, they implicitly coerced to strings, executing the payload.
**Learning:** Checking `typeof input !== 'string'` inside a leaf sanitizer is safe, but it requires the wrapper logic (`sanitizeObject`) to properly and deeply traverse structures (arrays, objects) to ensure strings nested inside other objects are successfully reached and sanitized. Blindly coercing everything to a string (`String(input)`) at the leaf level to fix this causes major data corruption (e.g. converting `[]` to `""`, or `{}` to `"[object Object]"`), which breaks application logic.
**Prevention:** Always implement a safe recursive object traverser (`sanitizeObject`) that accurately descends into arrays (`Array.isArray(obj)`) and objects (`typeof obj === 'object'`) before passing the scalar leaves to the string sanitizer (`sanitizeInput`).
