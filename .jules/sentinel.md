## 2025-02-27 - [CRITICAL] Type Confusion XSS & Email Header Injection in Quote Handlers
**Vulnerability:** A type confusion bypass allowed attackers to bypass the string sanitization function (`sanitizeInput`) by providing non-string inputs (like objects/arrays). Because they bypassed the regex replacements but were later coerced to strings via template literals in the HTML email body, an XSS vulnerability was introduced into the email output. Additionally, the email input lacked basic format validation and header values were not stripped of CRLF characters, making the backend vulnerable to Email Header Injection.
**Learning:** Checking `typeof input !== 'string'` as a fast-return path is dangerous if the output is later implicitly stringified. It creates a type confusion attack vector. Also, relying only on string sanitization for HTML injection doesn't protect against email protocol level vulnerabilities like Header Injection if line breaks aren't stripped from values used in subjects or replies.
**Prevention:** Ensure sanitization functions explicitly coerce all inputs to the expected type (e.g. `String(input)`) before performing sanitization, or strictly validate input types and reject others. Apply email format validation before processing and strictly filter `\r` and `\n` characters from variables passed into email headers (`subject`, `reply_to`).
