## 2024-04-26 - Email Header Injection via Unsanitized Subject and Reply-To Variables
**Vulnerability:** The contact form handlers (`functions/api/quote.ts` and `src/handlers/quote.ts`) used unsanitized user input (`data.name`, `data.type`, `data.email`) directly in the email `subject` and `reply_to` headers via the Resend API.
**Learning:** Even when basic XSS sanitization is applied, CRLF characters (`\r\n`) and invalid email formats can bypass these filters, allowing an attacker to inject arbitrary email headers or manipulate the email structure if the underlying email service doesn't strictly validate inputs. This is a common pattern when piping user forms to email APIs.
**Prevention:** Always strictly validate email fields using regex (e.g., `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`) before using them in `reply_to` or `to` headers. Always strip CRLF characters (`/[\r\n]/g`) from any user input that will be interpolated into an email header (like `subject`), regardless of other XSS sanitization.
