## 2025-02-28 - Type Confusion & CRLF Injection in Email Handler
**Vulnerability:** Input sanitization expected strings but didn't coerce input, allowing JSON arrays (e.g., `["<script>"]`) to bypass `.replace()` XSS filters (type confusion bypass). Additionally, the email address input was not validated for format or newlines, enabling CRLF header injection into the Resend API `reply_to` parameter.
**Learning:** Type-checking (`typeof input !== 'string'`) and returning early without coercion allows nested arrays to retain their original content and bypass filtering completely, leading to XSS when arrays are implicitly converted to strings during HTML concatenation. Additionally, unvalidated input mapped to email headers can lead to SMTP injection.
**Prevention:** Coerce inputs to strings explicitly (`String(input)`) before applying sanitation replaces, rather than short-circuiting on non-string types. Validate email formats rigorously and actively reject input containing CRLF characters (`\r\n`) before utilizing them as email headers or reply-to parameters.
