## 2024-04-17 - Prevent Header Injection and Enforce Strict Types in Quote API
**Vulnerability:** The Cloudflare Worker endpoint `functions/api/quote.ts` was vulnerable to header injection because user inputs (such as `name`, `type`, and `email`) were interpolated directly into email headers (e.g., `subject` and `reply_to` in the Resend API payload) without stripping CRLF characters (`\r`, `\n`). Additionally, the validation only checked for truthiness, allowing attackers to potentially bypass checks by sending arrays or objects instead of strings.
**Learning:** Even when using external APIs like Resend, input destined for headers must be strictly validated and sanitized to prevent injection attacks. Truthiness checks are insufficient for JSON payloads where the attacker controls the input type.
**Prevention:** Explicitly check for `typeof field === 'string'` during validation. Strip CRLF characters (`/[\r\n]/g`) from inputs before using them in headers. Add explicit regex format validation for specific fields like emails.
